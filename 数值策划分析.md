## ⚠️ 数值问题分析

### 问题1：进阶考试奖励过高 🔴 严重

**现状：**
- 进阶考试最高可获得26分
- 相当于审查2个发明的收益
- 导致进阶考试成为"必做项"

**影响：**
- 没做进阶题的学生起点差距过大
- 游戏前期就拉开巨大差距
- 降低后续游戏的公平性

**建议调整：**
```
简单题（1个原则）：3分 → 2分
中等题（2个原则）：5分 → 3分
困难题（3个原则）：8分 → 5分

最高总分：26分 → 16分
```

**调整理由：**
- 16分相当于1-2个审查的收益，合理
- 保持激励作用，但不至于差距过大
- 给未做进阶题的学生追赶机会

---

### 问题2：申诉成功奖励过高 🟡 中等

**现状：**
- 申诉成功审查官获得+25分
- 相当于2次正确审查的收益
- 可能导致过度申诉

**影响：**
- 学生可能为了高分而频繁申诉
- 申诉保证金5分相对奖励太低
- 游戏节奏被申诉打断

**建议调整：**
```
申诉保证金：5分 → 8分
申诉成功奖励：25分 → 20分
申诉失败（发明家）：5分 → 8分
```

**调整理由：**
- 提高申诉门槛，减少无意义申诉
- 降低申诉成功奖励，避免过度依赖申诉
- 保持申诉的战略价值

---

### 问题3：关联奖励可能被忽视 🟢 轻微

**现状：**
- 关联奖励2-3分
- 相对基础分12分较少
- 学生可能不重视

**影响：**
- 原则关联系统的教学价值未充分发挥
- 学生可能不会主动学习关联关系

**建议调整：**
```
保持现有数值，但增加视觉反馈：
- 识别关联时播放特效
- 在排行榜显示"关联大师"称号
- 统计关联识别次数
```

---

### 问题4：发明家收益单一 🟡 中等

**现状：**
- 发明家只有"骗过审查官"一种得分方式
- 收益：+12分/次
- 缺少其他激励

**影响：**
- 发明家角色吸引力不足
- 学生可能只想当审查官
- 游戏平衡性受影响

**建议增加：**
```
1. 创意奖励：
   - 发明被多人审查（说明有迷惑性）：+2分/人
   - 最多奖励：+6分（3人审查）

2. 辩护成功奖励：
   - 申诉被驳回时，发明家获得保证金：8分
   - 陪审团支持发明家：额外+3分

3. 连胜奖励：
   - 连续骗过3个审查官：+5分
   - 连续骗过5个审查官：+10分
```

---

### 问题5：游戏时长控制 🟡 中等

**现状：**
- 没有明确的游戏时长限制
- 可能导致课堂时间不够用

**建议：**
```
1. 设置游戏时长：
   - 大乱斗阶段：15-20分钟
   - 倒计时显示

2. 时间奖励机制：
   - 前5分钟：正常得分
   - 5-10分钟：得分×1.2
   - 10-15分钟：得分×1.5
   - 15分钟后：得分×2.0（冲刺阶段）

3. 最低参与要求：
   - 至少提交1个发明
   - 至少审查3个发明
   - 否则无法进入排行榜
```

## 📈 预期收益分析

### 30分钟游戏时长预估

#### 审查官路线（主打审查）
```
进阶考试：+10分（答对5题中的3题）
审查10次：
  - 7次正确（含2次关联）：+84分 + 6分 = +90分
  - 3次错误：-9分
  - 陪审团投票5次（3对2错）：+6分 - 4分 = +2分
总计：10 + 90 - 9 + 2 = 93分
```

#### 发明家路线（主打发明）
```
进阶考试：+10分
提交5个发明：
  - 3个成功骗过（每个被2人审查）：+36分 + 12分（创意）= +48分
  - 2个被识破：0分
  - 连胜3次奖励：+5分
审查5次（辅助）：
  - 3次正确：+36分
  - 2次错误：-6分
总计：10 + 48 + 5 + 36 - 6 = 93分
```

#### 平衡型路线
```
进阶考试：+10分
审查7次：
  - 5次正确：+60分
  - 2次错误：-6分
提交3个发明：
  - 2个成功：+24分 + 6分（创意）= +30分
总计：10 + 60 - 6 + 30 = 94分
```

**结论：** 三种路线收益基本平衡，鼓励多样化策略

## ⚖️ 平衡性检验

### 1. 角色平衡
- ✅ 审查官和发明家收益对等（都是12分基础）
- ✅ 两种角色都有额外奖励机制
- ✅ 鼓励学生同时扮演两种角色

### 2. 策略多样性
- ✅ 专精审查可行
- ✅ 专精发明可行
- ✅ 平衡发展可行
- ✅ 申诉流可行（但有风险）

### 3. 风险收益比
```
低风险低收益：
  - 只做基础审查：+12分/次
  - 稳定但慢

中风险中收益：
  - 提交发明：+12-24分/次
  - 看运气

高风险高收益：
  - 申诉：-8分 或 +20分
  - 需要判断力
```

### 4. 时间价值
```
每分钟期望收益：
  - 审查：3-4分/分钟（包括思考时间）
  - 发明：2-3分/分钟（需要创作时间）
  - 陪审：1-2分/分钟（等待时间长）

结论：审查效率最高，但发明更有趣
```

   ```javascript
   保证金：5分 → 8分
   成功奖励：25分 → 20分
   ```

### 第二阶段：增加发明家激励（推荐）

1. **创意奖励系统**
   ```javascript
   // 在 submitInventionReview 中添加
   if (invention.reviewCount === undefined) {
       invention.reviewCount = 0;
   }
   invention.reviewCount++;
   
   // 每被审查一次，发明家+2分（最多3次）
   if (invention.reviewCount <= 3) {
       users[inventorId].inventorScore += 2;
   }
   ```

2. **连胜奖励系统**
   ```javascript
   // 在用户数据中添加
   users[socket.id].inventorStreak = 0;
   
   // 骗过审查官时
   users[inventorId].inventorStreak++;
   if (users[inventorId].inventorStreak === 3) {
       users[inventorId].inventorScore += 5;
   }
   if (users[inventorId].inventorStreak === 5) {
       users[inventorId].inventorScore += 10;
   }
   
   // 被识破时重置
   users[inventorId].inventorStreak = 0;
   ```

### 第三阶段：游戏时长控制（可选）

1. **添加倒计时**
   ```javascript
   let gameStartTime = null;
   let gameDuration = 20 * 60 * 1000; // 20分钟
   
   // 开始大乱斗时
   gameStartTime = Date.now();
   
   // 定时检查
   setInterval(() => {
       const elapsed = Date.now() - gameStartTime;
       const remaining = gameDuration - elapsed;
       io.emit('gameTimer', { remaining });
       
       if (remaining <= 0) {
           gameState = 'finished';
           io.emit('gameState', gameState);
       }
   }, 1000);
   ```

2. **时间加成系统**
   ```javascript
   function getTimeMultiplier() {
       const elapsed = (Date.now() - gameStartTime) / 60000; // 分钟
       if (elapsed < 5) return 1.0;
       if (elapsed < 10) return 1.2;
       if (elapsed < 15) return 1.5;
       return 2.0; // 冲刺阶段
   }
   ```

   - 学生活跃时段

### 调整依据

```
如果审查官平均分 > 发明家平均分 × 1.2：

  → 提高保证金或降低奖励

  → 增加关联奖励或提示

如果游戏时长 > 30分钟：
```
