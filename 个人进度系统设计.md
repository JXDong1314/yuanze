# 个人进度系统设计 v3.0

## 核心概念

每个学生有自己独立的学习进度，根据积分自动或手动切换阶段。

## 学生状态结构

```javascript
{
    id: socket.id,
    name: "学生姓名",
    examinerScore: 0,        // 审查官分数
    inventorScore: 0,        // 发明家分数
    totalScore: 0,           // 总分 = examinerScore + inventorScore
    personalStage: 'learning', // 个人阶段
    hasCompletedLearning: false, // 是否完成学习
    qualificationPassed: false,  // 是否通过资格考
    currentTasks: [],
    reviewedInventions: []
}
```

## 个人阶段 (personalStage)

1. **learning** - 学习阶段
   - 初始状态
   - 学生学习12条原则和因果链
   - 完成学习后可进入考试

2. **qualification** - 资格考试阶段
   - 完成学习后进入
   - 基础考试（必做）+ 进阶挑战（可选）
   - 通过后获得初始积分，进入大乱斗

3. **battle** - 大乱斗阶段
   - 通过考试后进入
   - 可以发明创造和审查
   - 积分低于阈值时被踢回学习阶段

4. **failed** - 失败/重学阶段
   - 积分低于阈值（-10分）时进入
   - 必须重新学习和考试
   - 与learning阶段相同，但有警告提示

## 积分阈值规则

### 进入下一阶段的条件
- **learning → qualification**: 点击"完成学习"按钮
- **qualification → battle**: 通过基础考试（7/10题正确）
- **battle → failed**: 总分 < -10分（自动触发）
- **failed → qualification**: 重新完成学习

### 积分计算
```javascript
totalScore = examinerScore + inventorScore
```

### 阈值设置
```javascript
const SCORE_THRESHOLD = -10;  // 低于此分数需要重学
const PASS_THRESHOLD = 7;     // 基础考试通过分数
```

## 教师控制功能

### 1. 查看学生进度
- 显示每个学生当前所在阶段
- 显示学生积分
- 显示学生状态（学习中/考试中/战斗中/需重学）

### 2. 强制改变阶段
- **强制进入学习**: 让某个或所有学生回到学习阶段
- **强制进入考试**: 让某个或所有学生进入考试阶段
- **强制进入大乱斗**: 让某个或所有学生进入战斗（无视积分）
- **强制重学**: 惩罚某个学生，让其重新学习

### 3. 批量操作
- 全部进入学习
- 全部进入考试
- 全部进入大乱斗（已通过考试的）
- 重置所有进度

## 游戏流程

### 学生视角

```
登录 → 学习阶段 → 点击"完成学习" → 考试阶段 → 通过考试 → 大乱斗
                                                              ↓
                                                        积分 < -10
                                                              ↓
                                                        失败/重学 → 学习阶段
```

### 教师视角

```
开启游戏 → 监控学生进度 → 必要时强制调整阶段 → 结束游戏
```

## 事件系统

### 客户端 → 服务器

```javascript
// 学生完成学习
socket.emit('completeLearning');

// 学生提交考试
socket.emit('submitQualification', { type, answers });

// 教师强制改变学生阶段
socket.emit('adminForceStage', { userId, stage });

// 教师批量操作
socket.emit('adminBatchStage', { stage }); // 'learning', 'qualification', 'battle'
```

### 服务器 → 客户端

```javascript
// 更新个人阶段
socket.emit('updatePersonalStage', stage);

// 积分过低警告
socket.emit('scoreTooLow', { currentScore, threshold });

// 被踢回学习阶段
socket.emit('forcedToRelearn', { reason, currentScore });

// 阶段切换成功
socket.emit('stageChanged', { newStage, message });
```

## 实现要点

### 1. 积分监控
每次积分变化后检查：
```javascript
function checkScoreThreshold(userId) {
    const user = users[userId];
    const totalScore = user.examinerScore + user.inventorScore;
    
    if (totalScore < SCORE_THRESHOLD && user.personalStage === 'battle') {
        // 踢回学习阶段
        user.personalStage = 'failed';
        user.hasCompletedLearning = false;
        user.qualificationPassed = false;
        io.to(userId).emit('forcedToRelearn', {
            reason: '积分过低',
            currentScore: totalScore
        });
    }
}
```

### 2. 阶段切换验证
```javascript
function canEnterStage(user, targetStage) {
    switch(targetStage) {
        case 'qualification':
            return user.hasCompletedLearning;
        case 'battle':
            return user.qualificationPassed;
        default:
            return true; // learning 和 failed 可以随时进入
    }
}
```

### 3. 教师强制权限
教师可以无视规则强制改变阶段：
```javascript
socket.on('adminForceStage', (data) => {
    const { userId, stage } = data;
    if (users[userId]) {
        users[userId].personalStage = stage;
        io.to(userId).emit('updatePersonalStage', stage);
        io.emit('updateUserList', Object.values(users));
    }
});
```

## 优势

1. **个性化学习**：每个学生按自己的节奏学习
2. **激励机制**：积分高可以继续战斗，积分低需要重学
3. **灵活控制**：教师可以干预任何学生的进度
4. **自动管理**：系统自动检测积分并调整阶段
5. **公平竞争**：所有人都在同一个大乱斗池中竞争

## 注意事项

1. 积分阈值可以根据实际情况调整
2. 教师强制操作应该有确认提示
3. 学生被踢回学习阶段时应该有明确提示
4. 重学的学生应该保留之前的积分记录（用于统计）
