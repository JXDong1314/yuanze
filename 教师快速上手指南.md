# 教师快速上手指南

## 游戏流程概览

```
登录大厅 → 学习阶段 → 资格考试 → 大乱斗 → 结算排名
```

## 教师控制台操作步骤

### 1. 启动服务器
```bash
node server.js
```
访问：http://localhost:3000/admin

### 2. 游戏阶段控制

#### 第一阶段：开始学习阶段 📚
- **按钮**：蓝色 "📚 第一阶段: 开始学习阶段"
- **作用**：学生进入学习界面，可以：
  - 学习12条设计原则
  - 学习6条核心因果链
  - 查看游戏规则
- **建议时长**：10-15分钟

#### 第二阶段：开始资格考 🏁
- **按钮**：蓝色 "🏁 第二阶段: 开始资格考"
- **作用**：学生开始考试
  - 基础考试：10道单选题，答对7题通过
  - 进阶挑战：5道多选题，可获得初始分数（2-5分/题）
- **自动触发**：3人通过后自动进入大乱斗（可关闭）

#### 第三阶段：手动大乱斗 ⚔️
- **按钮**：绿色 "⚔️ 第三阶段: 手动大乱斗"
- **作用**：手动开启战斗阶段
- **使用场景**：关闭自动模式后使用

#### 强制开启大乱斗 ⚡
- **按钮**：橙色 "⚡ 强制开启大乱斗"
- **作用**：立即强制所有已通过基础考试的学生进入战斗
- **特点**：会关闭学生的进阶测试窗口
- **使用场景**：给学生一定时间做进阶测试后，统一开始

#### 第四阶段：荣耀结算 🏆
- **按钮**：黄色 "🏆 第四阶段: 荣耀结算"
- **作用**：结束游戏，显示最终排名

### 3. 其他功能

#### 自动开启大乱斗
- **开关**：复选框（默认开启）
- **作用**：3人通过资格考后自动进入大乱斗
- **建议**：小班教学建议关闭，大班教学建议开启

#### 重置游戏数据 🔄
- **按钮**：红色 "🔄 重置游戏数据"
- **作用**：清空所有数据，重新开始
- **警告**：会踢出所有学生

## 推荐教学流程

### 标准流程（45分钟课堂）
1. **准备阶段（5分钟）**
   - 学生登录系统
   - 教师检查在线人数

2. **学习阶段（10分钟）**
   - 点击"开始学习阶段"
   - 学生自主学习原则和规则
   - 教师可巡视答疑

3. **考试阶段（10分钟）**
   - 点击"开始资格考"
   - 学生完成基础考试（必做）
   - 学生选择进阶挑战（可选）

4. **战斗阶段（15分钟）**
   - 方式A：等待自动触发（3人通过后）
   - 方式B：5分钟后点击"强制开启大乱斗"
   - 学生发明创造和审查对抗

5. **结算阶段（5分钟）**
   - 点击"荣耀结算"
   - 查看排名和讨论

### 快速流程（30分钟）
1. 学生登录（3分钟）
2. 学习阶段（5分钟）
3. 资格考试（7分钟）
4. 大乱斗（12分钟）
5. 结算讨论（3分钟）

## 常见问题

### Q1: 学生看不到学习内容？
A: 确保点击了"开始学习阶段"按钮，学生才能看到学习界面。

### Q2: 什么时候用"手动大乱斗"，什么时候用"强制大乱斗"？
A: 
- **手动大乱斗**：关闭自动模式后，所有学生都准备好了再点击
- **强制大乱斗**：学生在做进阶测试，想统一开始时使用

### Q3: 如何让学生有更多时间做进阶测试？
A: 
1. 关闭"自动开启大乱斗"开关
2. 等待学生完成进阶测试
3. 点击"强制开启大乱斗"统一开始

### Q4: 申诉案件如何处理？
A: 三种方式：
- **执行民意**：按陪审团投票结果裁决（推荐）
- **强制驳回**：教师认为申诉无理
- **强制改判**：教师认为申诉有理

### Q5: 如何重新开始游戏？
A: 点击"重置游戏数据"按钮，所有数据清空，学生需要重新登录。

## 教学建议

### 核心教学目标
本游戏的核心教学目标是让学生理解**原则之间的因果关系**：
- 科学性 → 实用性
- 安全性 → 工程心理学
- 经济性 → 可持续性
- 实用性 → 可靠性
- 标准化 → 经济性
- 时效性 → 实用性

### 课后讨论要点
1. 哪些发明最有创意？
2. 哪些原则最容易被忽视？
3. 原则之间的因果关系如何体现？
4. 如何在实际设计中应用这些原则？

### 评分建议
- 审查官分数：反映对原则的理解深度
- 发明家分数：反映创意和对原则的灵活运用
- 综合评价：两个分数都重要

## 技术支持

### 服务器地址
- 学生端：http://localhost:3000
- 教师端：http://localhost:3000/admin

### 端口修改
如需修改端口，编辑 server.js 最后一行：
```javascript
server.listen(3000, () => {
    console.log('Server running on http://localhost:3000');
});
```

### 数据备份
游戏数据存储在内存中，重启服务器会清空。如需保存数据，请在结算后截图保存排名。
