# 更新说明 v3.0 - 个人进度系统

## 更新日期
2026年3月18日

## 重大变更

### 从全局阶段到个人进度
之前的版本使用全局游戏状态（所有学生同步），现在改为每个学生独立的个人进度系统。

## 核心特性

### 1. 个人阶段系统

每个学生有自己的学习进度，包含4个阶段：

#### 🎓 学习阶段 (learning)
- 初始状态
- 学生学习12条原则和6条因果链
- 完成后点击"完成学习，进入考试"按钮

#### 📝 考试阶段 (qualification)
- 完成学习后自动进入
- 基础考试（10题单选，7题通过）
- 进阶挑战（5题多选，可选，获得初始积分）
- 通过后自动进入大乱斗

#### ⚔️ 大乱斗阶段 (battle)
- 通过考试后自动进入
- 可以发明创造和审查
- 积分低于-10分时自动踢回学习阶段

#### ❌ 失败/重学阶段 (failed)
- 积分低于-10分时自动进入
- 显示当前积分和失败原因
- 必须重新学习和考试才能继续

### 2. 积分阈值机制

#### 自动降级
```
总分 = 审查官分数 + 发明家分数
如果 总分 < -10分 且 当前在大乱斗阶段
  → 自动降级到失败/重学阶段
```

#### 触发时机
每次积分变化后自动检查：
- 审查判定错误（-3分）
- 申诉失败（-8分）
- 陪审团投票错误（-2分）

### 3. 教师控制功能

#### 批量操作
- **全部进入学习阶段**：让所有学生回到学习阶段
- **全部进入考试阶段**：让所有完成学习的学生进入考试
- **已通过考试者进入大乱斗**：让所有通过考试的学生进入战斗
- **强制所有合格者进入大乱斗**：无视学生当前状态，强制进入
- **结束游戏并结算**：结束游戏，显示最终排名

#### 单个学生操作
每个学生旁边有三个按钮：
- **[学习]**：强制该学生进入学习阶段
- **[考试]**：强制该学生进入考试阶段
- **[大乱斗]**：强制该学生进入大乱斗（无视积分）

### 4. 学生进度监控

教师端显示每个学生的：
- 姓名
- 当前阶段（用图标和颜色标识）
- 审查官分数
- 发明家分数
- 总分（负分用红色显示）
- 快捷操作按钮

## 用户体验改进

### 学生端

#### 学习阶段
- 新增"完成学习，进入考试"按钮
- 学生自主控制学习节奏
- 不再需要等待教师开启考试

#### 考试阶段
- 通过考试后自动进入大乱斗
- 无需等待其他学生
- 可选进阶挑战获得初始积分

#### 大乱斗阶段
- 实时显示当前积分
- 积分过低时自动提示
- 被踢回学习时有明确说明

#### 失败/重学阶段
- 显示当前积分状态
- 说明失败原因
- 提供重新学习的指导
- 点击"开始重新学习"回到学习阶段

### 教师端

#### 学生列表增强
- 彩色阶段标识（蓝/黄/绿/红）
- 实时显示总分
- 负分学生背景变红
- 每个学生有独立操作按钮

#### 批量操作简化
- 按钮文字更清晰
- 操作前有确认提示
- 移除了"自动开启大乱斗"开关（不再需要）

## 技术实现

### 服务器端 (server.js)

#### 新增数据结构
```javascript
users[socketId] = {
    id, name,
    examinerScore, inventorScore,
    personalStage: 'learning', // 个人阶段
    hasCompletedLearning: false,
    qualificationPassed: false,
    currentTasks, reviewedInventions
}
```

#### 新增事件
- `completeLearning` - 学生完成学习
- `adminForceStage` - 教师强制改变单个学生阶段
- `adminBatchStage` - 教师批量操作

#### 新增函数
- `checkScoreThreshold(userId)` - 检查积分阈值
- `getStageDisplayName(stage)` - 获取阶段显示名称

### 客户端 (client.js)

#### 新增状态
```javascript
let myPersonalStage = 'learning';
```

#### 新增事件监听
- `updatePersonalStage` - 更新个人阶段
- `forcedToRelearn` - 被强制重学

#### 新增函数
- `completeLearning()` - 完成学习
- `restartLearning()` - 重新开始学习

### 界面 (index.html)

#### 新增界面
- `failed-screen` - 失败/重学界面
  - 显示当前积分
  - 说明失败原因
  - 提供学习建议
  - 重新开始按钮

#### 修改界面
- `learning-screen` - 添加"完成学习"按钮
- `screens` 对象 - 添加 failed 屏幕

### 管理端 (admin.js & admin.html)

#### 学生列表重构
- 显示个人阶段（图标+颜色）
- 显示总分
- 添加单个学生操作按钮

#### 新增函数
- `forceStage(userId, stage)` - 强制单个学生阶段
- `batchStage(stage, condition)` - 批量操作

## 游戏流程对比

### 旧版本（全局同步）
```
所有学生登录 → 教师开启学习 → 所有学生学习 
→ 教师开启考试 → 所有学生考试 
→ 教师开启大乱斗 → 所有学生战斗
```

### 新版本（个人进度）
```
学生A: 登录 → 学习 → 完成学习 → 考试 → 通过 → 大乱斗 → 积分-12 → 重学
学生B: 登录 → 学习 → 完成学习 → 考试 → 通过 → 大乱斗 → 积分+25 → 继续战斗
学生C: 登录 → 学习 → 完成学习 → 考试 → 未通过 → 重考 → 通过 → 大乱斗
```

## 优势

1. **个性化学习**：每个学生按自己的节奏学习
2. **激励机制**：积分高可以继续战斗，积分低需要重学
3. **灵活控制**：教师可以干预任何学生的进度
4. **自动管理**：系统自动检测积分并调整阶段
5. **公平竞争**：所有人都在同一个大乱斗池中竞争
6. **即时反馈**：学生立即知道自己的进度状态

## 注意事项

### 对教师
1. 不再需要等所有学生完成才能进入下一阶段
2. 可以随时查看每个学生的进度
3. 可以单独帮助进度落后的学生
4. 积分过低的学生会自动被踢回学习

### 对学生
1. 完成学习后要主动点击按钮进入考试
2. 注意保持积分在-10分以上
3. 被踢回学习阶段时要认真重学
4. 可以按自己的节奏学习，不用等其他人

## 兼容性

### 保留的功能
- 所有原有的游戏机制（发明、审查、申诉、陪审团）
- 所有原有的计分规则
- 所有原有的原则关系系统

### 移除的功能
- 全局游戏状态（gameState）
- 自动开启大乱斗开关（不再需要）
- qualifiedCount 计数器（不再需要）

## 升级指南

### 从 v2.x 升级到 v3.0

1. **备份数据**：虽然游戏数据在内存中，但建议先测试
2. **重启服务器**：`node server.js`
3. **清空浏览器缓存**：确保加载新的客户端代码
4. **测试流程**：
   - 学生登录 → 学习 → 完成学习 → 考试 → 大乱斗
   - 教师查看学生列表 → 尝试单个操作 → 尝试批量操作
   - 故意让学生积分降到-10以下 → 验证自动降级
