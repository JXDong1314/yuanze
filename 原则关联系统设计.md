# 设计原则关联系统设计方案

## 🎯 设计目标

让学生理解：
1. 某些原则经常一起被违背（强关联）
2. 某些原则互相矛盾（负关联）
3. 某些原则是独立的（弱关联）

---

## 📊 原则关联矩阵

### 强关联组（经常一起出现）

#### 组1：安全与人体工学
- **3.安全性** ↔ **8.工程心理学** 
- 例子：倒刺圆珠笔（不安全 + 不符合人体工学）
- 关联度：★★★★★

#### 组2：实用与可靠
- **4.实用性** ↔ **6.可靠性**
- 例子：纸质雨衣（不实用 + 不可靠）
- 关联度：★★★★☆

#### 组3：经济与可持续
- **5.经济性** ↔ **12.可持续发展**
- 例子：一次性黄金手机壳（不经济 + 不可持续）
- 关联度：★★★★☆

#### 组4：科学与实用
- **1.科学性** ↔ **4.实用性**
- 例子：方形轮子（不科学 + 不实用）
- 关联度：★★★★☆

#### 组5：时效与实用
- **9.时效性** ↔ **4.实用性**
- 例子：太阳能夜光手电筒（时效差 + 不实用）
- 关联度：★★★☆☆

#### 组6：标准化与经济
- **7.标准化** ↔ **5.经济性**
- 例子：三角形充电线（不标准 + 不经济）
- 关联度：★★★☆☆

### 矛盾关联（互斥原则）

#### 矛盾1：创新 vs 标准化
- **2.创新性** ⚡ **7.标准化**
- 说明：过度创新可能破坏标准，过度标准化限制创新
- 教学点：需要平衡

#### 矛盾2：经济性 vs 可靠性
- **5.经济性** ⚡ **6.可靠性**
- 说明：降低成本可能影响质量
- 教学点：需要权衡

#### 矛盾3：创新性 vs 安全性
- **2.创新性** ⚡ **3.安全性**
- 说明：新技术可能带来未知风险
- 教学点：创新需要安全验证

## 🎮 游戏机制设计

### 方案A：关联度加分系统（推荐）

#### 规则
当审查官正确识别出**强关联原则组**时，额外奖励：

```
基础分：12分
关联组合奖励：
- 2个强关联原则：+3分
- 3个强关联原则：+5分
- 包含矛盾关联：+2分（理解深度奖励）
```

#### 示例
**案例：倒刺圆珠笔**
- 标准答案：[3.安全性, 8.工程心理学]
- 审查官答对：12分（基础）+ 3分（关联组合）= **15分**

**案例：一次性黄金手机壳**
- 标准答案：[5.经济性, 12.可持续发展]
- 审查官答对：12分 + 3分 = **15分**

**案例：智能创新但不安全的产品**
- 标准答案：[2.创新性, 3.安全性]（矛盾关联）
- 审查官答对：12分 + 2分（理解矛盾）= **14分**

---

### 方案B：部分正确计分（更细致）

#### 规则
不要求完全匹配，按正确率计分：

```javascript
正确率 = (正确选中的数量 - 错误选中的数量) / 标准答案数量

得分 = 基础分 × 正确率

例子：
标准答案：[3, 8]（2个）
学生答案：[3, 8, 11]（选了3个，对2个，错1个）
正确率 = (2 - 1) / 2 = 50%
得分 = 12 × 50% = 6分
```

#### 优点
- 鼓励学生多思考
- 部分正确也有分数
- 减少"全对全错"的极端情况

#### 缺点
- 可能导致学生乱选（多选几个碰运气）
- 需要设置惩罚机制

---

### 方案C：原则链挑战模式（高级）

#### 规则
发明家可以设置"原则链"：

```
主要违背：[3.安全性]
次要违背：[8.工程心理学]（关联）

审查官得分：
- 只找到主要：8分
- 找到主要+次要：15分
- 找到次要但漏掉主要：3分
```

#### 优点
- 区分主次，更符合实际
- 鼓励深度思考

#### 缺点
- 复杂度较高
- 需要发明家有较高水平

---

## 💡 推荐实施方案

### 阶段1：基础模式（当前）
- 保持完全匹配机制
- 简单易懂，适合入门

### 阶段2：关联度加分（推荐添加）
- 在基础模式上增加关联组合奖励
- 不改变核心机制，只是额外奖励
- 鼓励学生发现原则之间的关系

### 阶段3：高级模式（可选）
- 部分正确计分
- 原则链挑战
- 适合高年级或进阶课程

## 🔧 技术实现（方案A）

### 1. 定义关联组

```javascript
// 在 server.js 中添加
const principleRelations = {
    // 强关联组（经常一起出现）
    strongGroups: [
        { ids: ['3', '8'], name: '安全与人体工学', bonus: 3 },
        { ids: ['4', '6'], name: '实用与可靠', bonus: 3 },
        { ids: ['5', '12'], name: '经济与可持续', bonus: 3 },
        { ids: ['1', '4'], name: '科学与实用', bonus: 3 },
        { ids: ['9', '4'], name: '时效与实用', bonus: 2 },
        { ids: ['7', '5'], name: '标准化与经济', bonus: 2 },
    ],
    
    // 矛盾关联（理解深度奖励）
    conflictPairs: [
        { ids: ['2', '7'], name: '创新vs标准化', bonus: 2 },
        { ids: ['5', '6'], name: '经济vs可靠', bonus: 2 },
        { ids: ['2', '3'], name: '创新vs安全', bonus: 2 },
    ]
};
```

### 2. 计算关联奖励

```javascript
function calculateRelationBonus(selectedAnswers, trueAnswers) {
    let bonus = 0;
    let bonusReasons = [];
    
    // 检查强关联组
    principleRelations.strongGroups.forEach(group => {
        const allInTrue = group.ids.every(id => trueAnswers.includes(id));
        const allInSelected = group.ids.every(id => selectedAnswers.includes(id));
        
        if (allInTrue && allInSelected) {
            bonus += group.bonus;
            bonusReasons.push(`识别出${group.name}关联 +${group.bonus}分`);
        }
    });
    
    // 检查矛盾关联
    principleRelations.conflictPairs.forEach(pair => {
        const allInTrue = pair.ids.every(id => trueAnswers.includes(id));
        const allInSelected = pair.ids.every(id => selectedAnswers.includes(id));
        
        if (allInTrue && allInSelected) {
            bonus += pair.bonus;
            bonusReasons.push(`理解${pair.name}矛盾 +${pair.bonus}分`);
        }
    });
    
    return { bonus, reasons: bonusReasons };
}
```

### 3. 应用到审查结果

```javascript
if (isExactMatch) {
    const relationBonus = calculateRelationBonus(selectedAnswers, trueAnswers);
    const totalScore = 12 + relationBonus.bonus;
    
    if (users[examinerId]) users[examinerId].examinerScore += totalScore;
    
    let msg = `慧眼识珠！判定完全正确！+${totalScore}分`;
    if (relationBonus.bonus > 0) {
        msg += `\n🎯 关联奖励：${relationBonus.reasons.join('、')}`;
    }
    
    io.to(examinerId).emit('reviewResult', { 
        success: true, 
        score: totalScore, 
        msg: msg,
        relationBonus: relationBonus,
        reviewId: reviewId, 
        inventionId: data.inventionId 
    });
}
```

## 📚 教学应用建议

### 课前准备
1. 讲解12条原则的基本含义
2. 展示原则关联矩阵
3. 分析经典案例中的原则关联

### 课中引导
1. 提示学生注意原则之间的关系
2. 鼓励寻找"组合拳"式的违背
3. 讨论矛盾原则的平衡点

### 课后总结
1. 统计最常见的原则组合
2. 分析学生容易忽略的关联
3. 讨论实际设计中的权衡

## 🎓 进阶玩法

### 1. 原则链挑战赛
- 发明家设计复杂产品，违背3-4个关联原则
- 审查官需要找出完整的原则链
- 高难度，高奖励

### 2. 矛盾平衡题
- 发明家设计"两难"产品
- 审查官需要指出哪些原则被牺牲，哪些被保留
- 培养权衡思维

### 3. 团队协作模式
- 3-4人一组，共同审查复杂案例
- 讨论原则关联，达成共识
- 培养团队协作能力

## ✅ 实施建议

### 第一步：添加关联度加分（简单）
- 不改变核心玩法
- 只是额外奖励
- 学生自然会注意到关联

### 第二步：显示关联提示（可选）
- 在界面上标注强关联组
- 例如：3和8用相同颜色标记
- 帮助学生建立认知

### 第三步：关联统计面板（进阶）
- 教师端显示关联统计
- 课后分析讨论
- 持续优化关联矩阵

## 🎯 预期效果

1. **深度理解**：学生不再孤立地记忆原则，而是理解原则之间的关系
2. **系统思维**：培养从多角度分析问题的能力
3. **实战能力**：更接近真实设计评审场景
4. **趣味性**：发现隐藏的关联组合，增加游戏深度
